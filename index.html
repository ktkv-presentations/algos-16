Композиция

Композиция

принцип объектно-ориентированного программирования (ООП), позволяющий создавать сложные объекты из более простых, используя объекты других классов в качестве полей

Пример

класс «Машина» может содержать объекты класса «Двигатель» или «Колесо», что отражает реальное представление о том, как машина состоит из отдельных компонентов.

Композиция и наследование

Наследование

отношение «is-a» (является), при котором класс-наследник является расширением или специализацией родительского класса

Композиция

отношение «has-a» (содержит), когда один класс включает в себя объекты других классов как свои компоненты

Пример композиции


class Address:
    def __init__(self, city, street):
        self.city = city
        self.street = street

    def __str__(self):
        return f'{self.street}, {self.city}'

class Person:
    def __init__(self, name, address):
        self.name = name
        self.address = address

    def __str__(self):
        return f'{self.name} lives at {self.address}'

# Создаем объект адреса
address = Address('Moscow', 'Tverskaya')

# Создаем объект человека, передав объект адреса
person = Person('John', address)

print(person)

Композиция в Python

реализуется за счет того, что внутри одного класса создаются объекты других классов 


# Класс для описания двигателя
class Engine:
    def __init__(self, power, fuel_type):
        self.power = power
        self.fuel_type = fuel_type

    def start(self):
        return f'Engine with {self.power} HP running on {self.fuel_type} started.'

# Класс для описания машины
class Car:
    def __init__(self, model, engine):
        self.model = model
        self.engine = engine  # Внедрение объекта Engine в класс Car

    def start_car(self):
        return f'Car model {self.model}: {self.engine.start()}'  # Использование методов объекта Engine

# Создание объекта двигателя
engine = Engine(150, 'Petrol')

# Создание объекта машины с двигателем
car = Car('Toyota', engine)

# Запуск машины
print(car.start_car())

# Car model Toyota: Engine with 150 HP running on Petrol started.

Перегрузка операторов

возможность определять или изменять поведение стандартных операторов (например, + , - , * , / , == и т.д.) для пользовательских объектов

Синтаксис методов перегрузки операторов:

  • __add__(self, other) — перегружает оператор сложения +
  • __sub__(self, other) - перегружает оператор вычитания -
  • __mul__(self, other) - перегружает оператор умножения *
  • __eq__(self, other) - перегружает оператор равенства ==
  • __lt__(self, other) - перегружает оператор «меньше» <
  • __gt__(self, other) - перегружает оператор «больше» >
  • __ne__(self, other) - перегружает оператор «неравенства» !=
  • __getitem__(self, index) - перегружает оператор доступа к элементам по индексу var[i]
  • __setitem__(self, index, value) - перегружает оператор «присваивания по индексу» var[i] = a
  • __len__(self) - перегружает оператор «больше» len()
  • len() - оператор длины >

Пример перегрузки оператора сложения + 


class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

# Создаем два вектора
v1 = Vector(2, 4)
v2 = Vector(3, 5)

# Используем перегруженный оператор сложения
v3 = v1 + v2
print(v3) # Vector(5, 9)

Пример перегрузки оператора сложения + 


class Matrix:
    def __init__(self, values):
        self.values = values

    def __mul__(self, other):
        result = [
            [self.values[i][j] * other for j in range(len(self.values[0]))]
            for i in range(len(self.values))
        ]
        return Matrix(result)

    def __repr__(self):
        return f"Matrix({self.values})"

# Создание матрицы
m = Matrix([[1, 2], [3, 4]])

# Умножение матрицы на скаляр
m2 = m * 2
print(m2) # Matrix([[2, 4], [6, 8]])
